         版本提交后，工作区与暂存区的内容不一致（即提交后，对工作区的内容进行了修改）

​

​ 将修改之后的工作区添加到暂存区之后（工作区与暂存区一致，与本地仓库不一致）

​

​ 将暂存区内容提交到本地仓库之后

​

​

​

  • 使用git checkout会使暂存区的内容覆盖工作区

  • 使用git reset会使本地仓库内容覆盖暂存区

  • git diff 比较工作区与暂存区的不同

  • git diff –cached 比较暂存区与本地仓库之间不同

git mv 旧名 新名 （更改文件名，并将改名后的文件放入暂存区）

暂存区和本地仓库相当于工作区的拷贝；

将工作区的内容用git add方法提交到暂存区就相当于把工作区的内容复制到暂存区

​ git commit 方法相当于把暂存区的内容复制到本地仓库

创建分支

只有master主分支当前本地仓库有版本时，才能创建并切换分支

在分支里，会复制一份master最新的本地仓库版本给该分支，在分支里的操作，只要没提交到本地仓库，

该分支在工作区的操作和在暂存区的操作是与master分支同步的；
