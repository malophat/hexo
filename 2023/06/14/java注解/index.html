java注解

发表于2023-06-14|更新于2023-06-14
|字数总计:608|阅读时长:1分钟|阅读量:

@Conctroller

用于设置资源路径@RequestMapping() 1.标注在类上。 2.标注在方法上。资源路径必须带 /

@RequestParam() 将请求参数与控制器方法的形参进行绑定三个值(value，required<—defaultValue(默认值)）

用于设置资源路径@WebServlet(“/sr1”)，资源路径必须带 /，如果是(“/“)会自动访问，相当与SpringMVC下的index文件

占位符

mybatis中的#{}，${}

thymeleaf渲染html中的@{}

@RestController是@Controller和 @ResponseBody 的结合体，两个标注合并起来的作用。@RestController类中的所有方法只能返回String、Object、Json等实体对象，不能跳转到模版页面。

@RestController中的方法如果想跳转页面，则用ModelAndView进行封装，如下：

ModelAndView mv = new ModelAndView(“index”);
return mv;

@qualifier()

@Value(“${}”)用于获取yml配置文件中的属性值，如果与配置文件中的属性名不一致则抛出异常

@Value注解参数中如果不加${}则会被识别为字符串，但会根据声明的变量类型自动进行类型转换，如果无法转换则会直接抛出异常

在yml中，属性值想使用转义字符，需要使用双引号包裹

如果配置文件中属性值是一个数字，并且该数字以0或0x等标识开头，在属性注入的时候会被进制转换

yml或properties的配置文件，在Springboot中是Spring中的Environment类的一个bean，可以@Autowired自动装配到Environment类的对象中

  • springboot单体项目中页面放置在resources/static目录下
  • static下的资源可以直接访问，访问路径是应用程序上下文+要访问的页面

2.@Scope注解怎么使用
 直接在bean对象方法上增加@Scope注解即可。

注入配置文件信息方式：1. @Value(“${}”) 2.ConfigurationProperties(prefix=””)

(注入配置文件信息中ConfigurationProperties中prefix参数的值可以松散（匹配范围大）)；

自动注入Spring容器中bean对象的方式：@Autowired

@SpringbotTest() ：参数1.classes=(启动类的字节码文件)，2.properties={“键=值”}配置文件中的属性键值对，来临时修改配置文件中的属性值

@bean注解中可能会遇到的问题，加入方法返回的对象的类是DogFactoryBean,该factory实现了接口FactoryBean，并且指定了泛型，那么new DogFactoryBean()其实就是new Dog()对象，这样的好处是在new DogFactoryBean()时可以自定义一些初始化工作
